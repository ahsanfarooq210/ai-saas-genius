# Change records

Files in this folder are dated implementation records. They preserve why a behavior changed, tests added at that time, and rollback context.

They are not the main description of the current system. For current behavior, use:

- [`../handbook/`](../handbook/)
- [`../current/`](../current/)
- [`../graphs/`](../graphs/)
- [`../persistence/`](../persistence/)

When a new change record is added, also update the relevant current reference. Do not expect a reader to reconstruct today's backend by reading change notes chronologically.
